## Project Overview

Nabbihni is THEBOLDS' Arabic-first social countdown app for iOS and Android. Users create personal countdowns (birthdays, milestones, deadlines) and browse 95+ Saudi public events (Ramadan, Eid al-Fitr, Eid al-Adha, National Day, etc.) with Hijri calendar support, celebration animations, deep link sharing, and home screen widgets on both platforms. It is THEBOLDS' first revenue-generating consumer app, targeting 35M+ Arabic speakers in Saudi Arabia with a freemium model: 5 free countdowns with halal-filtered ads, or premium (4.99 SAR/month or 79.99 SAR lifetime) for unlimited countdowns, no ads, and premium themes. Revenue is handled via RevenueCat (subscriptions) and Google AdMob (ads).

## Architecture

```
nabbihni/
├── app/                    # expo-router screens
│   ├── (tabs)/
│   │   ├── index.tsx       # Home (my countdowns)
│   │   ├── explore.tsx     # Public countdowns
│   │   └── settings.tsx    # Settings
│   ├── countdown/
│   │   ├── [id].tsx        # Countdown detail
│   │   └── create.tsx      # Create countdown
│   └── _layout.tsx         # Root layout
├── components/             # Reusable UI components (AdBanner, etc.)
├── contexts/               # React contexts (SubscriptionContext)
├── hooks/                  # Custom hooks (useCountdown, useCountdowns)
├── lib/                    # Utilities (storage, widgetData)
├── constants/              # App constants (themes, publicEvents)
├── types/                  # TypeScript type definitions
├── widgets/                # Android widget components
├── targets/                # iOS widget (SwiftUI via @bacons/apple-targets)
├── plugins/                # Expo config plugins (withNoSplashLogo, etc.)
├── locales/                # ar.json, en.json
├── supabase/               # Supabase migrations and config
├── ios-storekit/           # StoreKit configuration for testing
├── marketing/              # Marketing assets
├── docs/                   # Documentation
├── scripts/                # Build/dev scripts
├── metro.config.js         # Metro config (stubs native modules in dev)
├── app.json                # Expo/EAS config
├── eas.json                # EAS Build config
└── store.config.json       # Store listing metadata
```

**Key files:**
- `hooks/useCountdown.ts` — Timer logic and formatting
- `hooks/useCountdowns.ts` — CRUD operations for countdowns
- `lib/storage.ts` — AsyncStorage helpers
- `lib/widgetData.ts` — Widget data sync layer
- `constants/themes.ts` — Visual themes (5 free)
- `constants/publicEvents.ts` — Saudi Arabia events (95+)
- `contexts/SubscriptionContext.tsx` — RevenueCat integration
- `components/AdBanner.tsx` — AdMob component
- `metro.config.js` — Native module stubbing (critical for dev builds)

## Key Decisions

| Decision | Rationale | Decided By |
|----------|-----------|------------|
| React Native + Expo | Cross-platform, shared stack with Mutmaeen | Mohammad |
| RevenueCat for payments | Industry standard, handles both stores, receipt validation | Mohammad |
| Freemium with halal ads | Low barrier to entry, proven mobile model, Shariah-compliant | Mohammad + Business dept |
| Arabic-first (RTL) | Underserved market, cultural advantage | Mohammad |
| SAR pricing (4.99/mo, 79.99 lifetime) | Target Saudi market, local currency reduces friction | Business dept |
| No splash image | Explicit request by Mohammad; enforced via withNoSplashLogo plugin | Mohammad |
| New Architecture enabled | Required by react-native-reanimated 4.x on RN 0.81+ | Engineering |
| Metro stub native modules in dev | Prevents crashes from unlinked native code; skip only in EAS builds | Engineering |
| Never auto-reload for RTL | forceRTL needs native restart; JS reload causes infinite loop | Engineering |
